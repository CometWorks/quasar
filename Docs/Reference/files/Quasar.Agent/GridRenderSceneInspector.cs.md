# Quasar.Agent/GridRenderSceneInspector.cs

**Module:** Quasar.Agent  **Kind:** class  **Tier:** 1

## Summary
Game-thread helper that builds viewer scene snapshots from live `MyCubeGrid` and asteroid `MyVoxelBase` instances. It maps primary and optional context grids into `Magnetar.Protocol` viewer DTOs, computes selected-grid-local context bounds plus matching relative/world AABBs and explicit snapped 3D clip-prism bounds, enumerates loaded intersecting grids deterministically, expands every shown grid through mechanical connections so subgrids render fully even outside context clipping, selects surrounding grids, blocks, voxel bodies, and voxel samples by projecting their bounds into the selected grid's relative view space and testing the same snapped X/Y/Z prism used by browser clipping, registers active scene mod asset roots and logical model/texture names with mod root hints, includes generated cube-part and runtime subpart metadata with subpart entity IDs, refreshes and exports captured emissive material states including light-specific render-property bulb updates, captures current-model orientation matrices plus build level, integrity, max integrity, accumulated damage, and damaged-block skeleton offsets, captures scene-wide conveyor/logistics topology and conveyor reachability so online links across visible mechanical subgrids share highlight systems without marking context-clipped but still connected endpoints as dangling, captures terminal-synced lighting and LCD metadata, assigns grid ownership on blocks/chunks/lights, and emits separate voxel damage chunks with per-sample masks produced by comparing current storage against the voxel data provider's original terrain.
